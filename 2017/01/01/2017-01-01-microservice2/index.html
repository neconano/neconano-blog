轻量级微服务架构再读

传统的web架构

巨石一块web服务（ui+业务模块），使用lb（负载均衡）进行水平扩展，数据库集群（缓存，读写分离）

面临的问题

1.资源浪费：每个副本都是完整的一块巨石

2.难于部署：每个巨石必须完整，对于单一java架构会难维护

微服务

微服务架构模式

微服务需要满足的要求

RPC ： 使用某种协议（mq,tcp,http..）调用目标接口（一般是长链接，开销小），获得结果的过程

开发流程

架构师：拆分产品为若干服务，为每个服务设计API接口，给出API文档（名称，版本，请求参数，响应结果，错误代码等）

开发工程师：实现API接口，完成API单元测试

前段工程师：并行开发UI部分，根据API文档模拟数据完成开发

开发规范，使用GitFlow工作流

该规范根据实际情况适用即可

1.有效的监控

安装docker

10张图带你深入理解Docker容器和镜像

centos环境

yum -y install docker

yum -y install docker-registry

windows环境(不推荐)

可以选择windows版本

默认情况下，Docker守护进程会生成一个socket（/run/docker.sock）文件来进行本地进程通信，而不会监听任何端口，因此只能在本地使用

配置 /etc/sysconfig/docker

-H unix:///run/docker.sock -H 0.0.0.0:2375

安装maven

‘版本目录’中选择一下载

添加镜像（安装目录的config/setting.xml）

生成docker镜像

mvn clean package

mvn docker:build

centos环境需要安装maven和docker并启动，intellij idea项目下执行即可

推送镜像

运行docker registry服务,

参考

docker run -d -p 5000:5000 –name registry registry:2

当前的版本应该是v2，查询使用下面的命令

http://10.168.1.215:50000/v2/_catalog

修改配置(默认https)

执行mvn docker:push

提示错误 ： http: server gave HTTP response to HTTPS client

拉取镜像

本地是不需要配置证书的

使用mvn docker:push的话应该是需要配置证书

推送镜像 docker push 10.168.1.215:50000/demo.msa/msa-framework

拉取镜像 docker pull 10.168.1.215:50000/demo.msa/msa-framework:1.0-SNAPSHOT

需要注意tag，也就是版本信息，否则会提示找不到

运行镜像

docker run -d -p 8080:8080 镜像id

docker pull s-01:50000/demo.msa/msa-framework

openssl genrsa -aes25 -out ca.pem 1024

//key
openssl genrsa -out ca.key 1024

//根证书签发申请文件(csr文件)
openssl req -new -key ca.key -out ca.csr -subj \
“/C=CN/ST=Shanghai/L=Shanghai/O=Test/OU=Docker/CN=s-01”

//自签发根证书(cer文件)
openssl x509 -req -days 365 -sha1 -extensions v3_ca -signkey \
ca.key -in ca.csr -out ca.cer

//server端证书请求文件
openssl req -new -key ca.key -out ca-srv.csr -subj \
“/C=CN/ST=Shanghai/L=Shanghai/O=Test/OU=Docker/CN=s-01”

//使用根证书签发服务端证书
openssl x509 -req -days 365 -sha1 -extensions v3_req -CA ca.cer -CAkey ca.key \
-CAserial ca.srl -CAcreateserial -in ca-srv.csr -out ca-srv.cer

//crt证书
openssl req -new -x509 -days 7300 -key ca.key -out ca-srv.crt -subj \
“/C=CN/ST=Shanghai/L=Shanghai/O=Test/OU=Docker/CN=s-01” \
-config /etc/pki/tls/openssl.cnf

docker run -d -p 5000:5000 \
-v /etc/docker/certs.d/s-01:/certs \
-e REGISTRY_HTTP_TLS_CERTIFICATE=/certs/ca-srv.crt \
-e REGISTRY_HTTP_TLS_KEY=/certs/ca.key \
registry:2

//client端证书请求文件
openssl req -new -key ca.key -out ca-clt.csr -subj \
“/C=CN/ST=Shanghai/L=Shanghai/O=Test/OU=Docker/CN=s-01”

//使用根证书签发客户端证书
openssl x509 -req -days 365 -sha1 -extensions v3_req -CA ca.cer -CAkey ca.key \
-CAserial ca.srl -in ca-clt.csr -out ca-clt.cer

//服务器证书
cat ca-srv.crt ca.key > ca.pem
