tp-uc笔记

1.框架通信原理

1.UC通信仅与UC库产生数据关系

2.通信配置：
A论坛需要配置config_ucenter.php;apps.php;config.inc.php;
B框架配置uc.php

并且​config.inc.php中的UC_KEY等没什么关系，只和数据库配置有效

3.通信分发原理：通信成功后即可使用UC进行登录分发。分发调用uc_user_synlogin函数，框架与论坛均有该函数，均可以通信分发。

4.登录原理：分发后利用论坛生成作用域cookie，论坛随即登录，框架需要模拟论坛解析cookie登录

2.问题处理

A.后台通信失败

1.检查域名（通信地址）是否可用

B.Access denied for agent changed 错误

1、UCenter 应用 ID 需要和 UCenter 后台的 ID 一致；

2、UCenter 通信密钥需要和 UCenter 后台的通信密钥一致；

3、UCenter 访问地址查看是否是正确的地址；

4、UCenter IP 地址查看是否是UCenter所在的 Mysql 服务器的ip地址。

5、应用IP：尝试设置为该应用所在服务器的 IP 地址，本地设置为127.0.0.1

6、分发的客户端要与UC服务器首先通信成功

7、最后，看发起通信的框架是什么，则有可能是该框架配置有问题
