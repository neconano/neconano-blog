s服务器简单同步样例

1.代码同步配置

服务端：121.41.49.233

1.创建主配置文件”rsyncd.conf”并写入标准配置参数

vim /etc/rsyncd.conf

“##rsync server##
uid = www
gid = www
use chroot = no
max connection = 2000
timeout = 600
pid file = /var/run/rsync.pid
lock file = /var/run/rsync.lock
log file = /var/log/rsyncd.log
ignore errors
read only = false
list = false
hosts allow = 10.47.68.95
hosts deny = 0.0.0.0/32
auth users= rsync_backup
secrets file= /etc/rsync.password

#############################
[backup1]
comment = izaodao
path = /newdata/www/“

2.创建密码文件

echo “rsync_backup:Rsyncizaodao2016” > /etc/rsync.password

chmod 600 /etc/rsync.password

ll /etc/rsync.password

cat /etc/rsync.password

3.启动rsync

rsync –daemon

ps -ef|grep rsync

netstat -lntup|grep rsync

4.加入开机自启动

echo “rsync –daemon” >> /etc/rc.local

4.编辑测试脚本

vim /usr/jiaoben/inotify-1.sh

写入以下内容：

“#!/bin/bash
inotify=/usr/local/inotify-tools/bin/inotifywait
$inotify -mrq –format ‘%w%f’ -e create,delete,modify,attrib,move,close_write /newdata/www/www.izaodao.com/ \
|while read file
do

rsync -az /newdata/www/www.izaodao.com –delete –exclude-from=/usr/jiaoben/exclude.list \
rsync://rsync_backup@10.168.176.99/backup1/ –password-file=/etc/rsync.password
done”

vim /usr/jiaoben/exclude.list <==创建排除文件,并写入需要排除的目录

写入一下内容：

“data
zdschool/data
uc/data”

vim /usr/jiaoben/inotify-2.sh

写入以下内容：

“#!/bin/bash

|while read file
do

rsync -az /newdata/www/ServiceSystem –delete rsync://rsync_backup@10.168.176.99/backup1/ –password-file=/etc/rsync.password
done”

2.NFS配置

【服务端232配置】

1.安装软件

yum install nfs-utils rpcbind -y

2.启动服务

/etc/init.d/rpcbind start

rpcinfo -p localhost

/etc/init.d/nfs start

rpcinfo -p localhost

(提示：启动顺序很重要应最先启动rpcbind)

3.设置开机自启动

chkconfig rpcbind on

chkconfig nfs on

4.配置nfs服务

id www

uid=501(www) gid=501(www) groups=501(www)

vim /etc/exports <==编辑主配置文件

添加如下内容:

“/newdata/www/www.izaodao.com/zdschool/data 10.168.176.99(rw,sync,all_squash,anonuid=501,anongid=501,insecure)
/newdata/www/www.izaodao.com/uc/data 10.168.176.99(rw,sync,all_squash,anonuid=501,anongid=501,insecure)
/newdata/www/www.izaodao.com/data 10.168.176.99(rw,sync,all_squash,anonuid=501,anongid=501,insecure)
“

(解释：rw读写, sync将数据同步写入内存缓冲区与磁盘中, all_squash将远程访问的用户身份全部映射成匿名用户默认为nfsnobody, insecure允许客户端从大于1024的TCP/IP端口连接服务器)

anonuid指定映射的匿名用户需要使用该用户UID例如”anonuid=501”,anongid指定GID (这两个参数使用的前提是”all_squash”)

cat /var/lib/nfs/etab <==可查看nfs默认使用的用户及共享参数

5.重新加载服务

/etc/init.d/nfs reload

6.检查和测试挂载

showmount -e localhost

【客户端233配置】

1.安装软件

yum install nfs-utils rpcbind -y

(提示：客户端安装nfs-utils是为了可以使用showmount)

2.启动服务

/etc/init.d/rpcbind start

3.设置开机自启动

4.测试服务端共享情况

5.挂载

mount -t nfs 10.47.68.95:/newdata/www/www.izaodao.com/zdschool/data /newdata/www/www.izaodao.com/zdschool/data

mount -t nfs 10.47.68.95:/newdata/www/www.izaodao.com/uc/data /newdata/www/www.izaodao.com/uc/data

mount -t nfs 10.47.68.95:/newdata/www/www.izaodao.com/data /newdata/www/www.izaodao.com/data
