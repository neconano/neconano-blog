方案1.重启下系统

sudo apt-get update
sudo apt-get install openssh-server

输入”sudo apt-get install vsftpd”
判断vsftpd是否安装成功
sudo service vsftpd restart
让root用户远程登陆,注释掉 /etc/ftpusers 中的root即可
上傳文件時總是出現550 Permission denied錯誤，
查看vsftpd的配置文件sudo vim /etc/vsftpd.conf,
發現write_enable=YES這句配置項默認是被註釋掉的即#write_enable=YES,
把前面的註釋去掉，重啓vsvfpd，一切正常了。

vi /etc/profile

export JAVA_HOME=/usr/lib/jvm/default-java
export JRE_HOME=$JAVA_HOME/jre
export CLASSPATH=.:$CLASSPATH:$JAVA_HOME/lib:$JRE_HOME/lib
export PATH=$PATH:$JAVA_HOME/bin:$JRE_HOME/bin
退出生效
source /etc/profile
验证
echo $JAVA_HOME
