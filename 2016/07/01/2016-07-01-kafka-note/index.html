3.集群安装kafka

启动kafka,在每个节点上执行

nohup ./bin/kafka-server-start.sh config/server.properties &

创建一个topic,任意节点执行

bin/kafka-topics.sh –create –zookeeper 192.168.1.197:2181 –replication-factor 3 –partitions 1 –topic my-topic

查看创建的topic,任意节点执行

bin/kafka-topics.sh –list –zookeeper 192.168.1.199:2181 my-topic

4.相关概念

生产者(Producers)

生产者往某个Topic上发布消息。生产者也负责选择发布到Topic上的哪一个分区。最简单的方式从分区列表中轮流选择。也可以根据某种算法依照权重选择分区。开发者负责如何选择分区的算法。

消费者(consumer)扩展阅读

Kafka的consumer是以pull的形式获取消息数据的。不同于队列和发布-订阅模式，kafka采用了consumer group的模式。通常的，一般采用一个consumer中的一个group对应一个业务，配合多个producer提供数据。

5.常见问题

官方faq

  1. ERROR [ReplicaFetcherThread-0-3], Error for partition [t2,2] to broker 3:org.apache.kafka.common.errors.UnknownServerException: The server experienced an unexpected error when processing the request

原因：多台broker使用了相同的zookeeper地址(集群中的一台broker的配置文件server.properties中添加了参数host.name=127.0.0.1导致)

解决：登陆集群中的一台zookeeper客户端，使用命令（get /brokers/ids/1）查看broker的注册信息,有问题的一台broker信息为host:127.0.0.1,根据id号

找到对应broker,调整配置文件server.properties，重启kafka解决
